# NPL→RWA 토큰화 — 앵커투자자 GTM 수요 전략

> 작성: 2026-06-24  
> 분류: 내부 전략 문서 (외부 배포 금지)  
> 배경: CEO 검토 결론 — "공급(7.7조) 준비됐으나 수요 파이프라인 없음 = 발사대 없는 로켓"  
> 참조: NPL_RWA_사업계획서_260624.pdf §2·§3·§8, docs/npl-rwa-tokenization.md, docs/npl-regulatory-sandbox.md

## 핵심 전제

**"일반 리테일 투자자"는 지금 당장의 수요가 아니다.**  
NPL 토큰은 회수 불확실성, 긴 만기(6~24개월), 담보 구조 이해를 요구하는 상품이다.  
1호 발행은 "이 구조를 이미 이해하고 있는 사람 20~50명"이 채워야 한다.  
수요 전략의 출발점은 그 20~50명이 누구인지 특정하는 것이다.

## 1. 타겟 투자자 세그먼트 정의

### 세그먼트 우선순위 표

| 순위 | 세그먼트 | 왜 NPL 토큰을 사는가 | 최소 티켓 | 리스크 감내도 | 접근 경로 | 우선순위 |
|------|----------|---------------------|-----------|--------------|-----------|---------|
| ★★★ | NPL 업계 내부자 — AMC·경·공매 전문 투자자·법무법인 관계자 | 이미 NPL 구조를 안다. 기존에는 억 단위 진입해야 했는데 소액 참여 가능 | 1,000만~5,000만 | 높음 (업무상 익숙) | 기존 거래 네트워크, NPL 컨퍼런스 | **P0** |
| ★★★ | 패밀리오피스·고액자산가 중 부동산·NPL 경험자 | 부동산 직접 투자 대비 관리 부담 없이 유사 IRR 확보. 조각화로 분산 가능 | 5,000만~3억 | 높음 | PB 채널, 부동산 패밀리오피스 네트워크 | **P0** |
| ★★ | 적격투자자 엔젤·전문투자자 (스타트업 투자 경험자) | 실물 담보 기반 고정 IRR 자산으로 포트폴리오 방어선 구축 | 3,000만~1억 | 중~높음 | 엔젤클럽, 스타트업 IR 행사 접점 | **P1** |
| ★★ | 평가 SaaS 고객 — NPL 투자사·중소 AMC·법무법인 | 플랫폼 데이터로 상품을 알고 있음. SaaS 사용자 → 청약자 전환 | 1억~10억 | 높음 (기관) | SaaS 영업 채널 내부 전환 | **P1** |
| ★ | 기관 LP의 소액·실험적 배분 창구 (연기금·공제회 산하 실험 포트) | ESG·대체투자 다변화 의무 이행. 소액 파일럿으로 내부 승인 부담 낮춤 | 10억~50억 | 중 (내부 승인 필요) | GP 네트워크, 대체투자 컨퍼런스 | **P2** |
| — | 일반 리테일 투자자 | 조각투자 플랫폼 UX에 익숙하나 NPL 구조 이해 없음 | 10만~100만 | 낮음 | 앱 마케팅 | **지금은 아님** |

### 세그먼트별 Dream Outcome & Pain

| 세그먼트 | Dream Outcome (진짜 원하는 것) | Current Pain (NPL 투자의 고통) |
|----------|-------------------------------|-------------------------------|
| NPL 내부자 | 자본 없이 딜 참여. 수수료 말고 수익 지분 | 억 단위 최소 진입, 현금 묶임 12~36개월 |
| 패밀리오피스·HNWI | 부동산 직투 IRR에 관리 부담 없이 접근 | 경·공매 낙찰 리스크, 명도 문제, 대리인 비용 |
| 엔젤·적격투자자 | 스타트업 대비 실물 담보로 원금 방어 | NPL은 기관 전용이라 참여 채널 자체가 없었음 |
| SaaS 고객(AMC·법무) | 플랫폼 데이터로 이미 알고 있는 물건에 직접 투자 | 내부 심사 통과해도 자본 소진, LP 모집 별도 필요 |
| 기관 LP 실험창구 | 대체투자 비중 채우기 + 내부 성과 데이터 확보 | 기존 NPL 펀드는 최소 50억, 2~3년 Lock-up |

**솔직한 평가:**  
P0 두 세그먼트(내부자·HNWI)가 1호 발행의 현실적 앵커다. 나머지는 파이프라인이 되려면 6개월 이상 관계 구축이 필요하다.

## 2. 1호 토큰 발행 — 청약 시나리오

### 2.1 담보 유형 선택 — "이해하기 쉽고 회수 명확한 물건"

CEO 검토 원칙을 그대로 적용한다: 지방 토지 NPL은 1호에 올리지 않는다.

| 조건 | 1호 권고 기준 |
|------|--------------|
| 담보 유형 | **수도권 아파트 담보 NPL** — 낙찰가율 85~92%, IRR 15~25%, 회수기간 6~12개월 |
| 물건 수 | 3~5건으로 소규모 풀 구성. 단일 물건은 집중 리스크 |
| 채권 액면 규모 | OPB 기준 총 20~50억 (실 매입가 15~40억) |
| 회수 명확성 | 경매 진행 중이거나 낙찰 직전 단계 물건 우선 — "회수 가시성"이 청약 설득의 핵심 |
| 배제 항목 | 지방 토지, 상가, 위반건축물 등재 물건, 다수 임차인 복잡 물건 |

**이유:** 수도권 아파트는 투자자가 시세를 직접 조회할 수 있다. p10~p90 신뢰구간이 낯설어도 "강남구 아파트, KB시세 8억"은 이해된다. 회수 논리를 투자자가 독립 검증 가능한 것이 1호 담보의 조건이다.

### 2.2 발행 규모·티켓 가설

| 항목 | 가설값 | 근거 |
|------|--------|------|
| 총 발행 규모 | 20억~40억 | Y2 RWA 수익 80억 목표 → 1호는 파일럿. 규모보다 완주가 중요 |
| 토큰 단가 | 100만 원/토큰 | HNWI·내부자 티켓에 맞춤. 5만 원 단위는 리테일용으로 나중 |
| 최소 청약 | 10토큰 (1,000만 원) | 적격투자자 기준 부합, 의미 있는 규모감 |
| 목표 투자자 수 | 20~40명 | 사모 50인 미만 유지 → 공모 증권신고서 없이 파일럿 가능 (법무 확인 필요 [T2]) |
| 예상 IRR | 15~20% | 수도권 아파트 기준, 비용 차감 후 보수 추정 |

### 2.3 앵커 → Fill 구조

```
[앵커 1~2곳] → 총 발행의 50~60% 선청약 확보
    │
    └─ 앵커 조건: 발행 전 LOI(의향서) 접수 + 구조 사전 리뷰 참여
                  대가: 우선 배정권 또는 정보 우선 제공

[나머지 40~50%] → 기존 네트워크(엔젤·SaaS 고객·HNWI) 대상 청약
    │
    └─ 앵커 선청약 확인 후 공개 → "이미 절반 찼다"는 사실이 사회적 증거로 작동
```

**앵커 없이 일반 청약부터 받으면 실패한다.** 첫 번째 청약자가 될 용기를 가진 개인은 없다.  
앵커 확보가 1호 발행의 선결 조건이며, 이것이 수요 파이프라인이 가장 먼저 해야 할 일이다.

## 3. 청약의향 명단(Pipeline) 빌드 전략

### 3.1 의향명단 소스 4개 채널

| 채널 | 방법 | 예상 타겟 수 | 기간 |
|------|------|------------|------|
| **기존 NPL 거래 네트워크** | 지금까지 직접 경·공매·AMC와 거래한 관계자에게 "토큰화 파일럿 사전 청약 의향" 타진 | 10~30명 | 즉시 시작 |
| **평가 SaaS 초기 고객** | SaaS 영업 과정에서 접촉한 NPL 투자사·법무법인 담당자 → 청약 의향 전환 | 5~15명 | 즉시 시작 |
| **부동산·NPL 특화 세미나** | 내부 주최 소규모 세미나 (20~30명) — 토큰화 구조·IRR 백테스트 발표 → 명함 수집 [T2] | 30~50명 | 3개월 내 |
| **적격투자자 네트워크** | 엔젤클럽·투자모임 발표 요청 [T2] | 10~20명 | 3~6개월 |

### 3.2 의향 단계 정의 (Funnel)

```
인지 → 관심 → 정보 요청 → LOI(의향서) → 청약
  |       |         |            |           |
  └ 세미나  └ 1:1 미팅  └ IM 발송   └ 구조 리뷰  └ KYC 완료
    소개글    (30분)     (투자안내문)    참여        입금
```

- **LOI 단계부터 본격 파이프라인으로 분류**. 관심·정보요청은 예비 파이프라인.
- LOI는 법적 구속력 없는 의향 확인서 형태로 운영 (1~2페이지, 투자금액·시기 기재).

### 3.3 CRM 관리 필드

| 필드 | 설명 |
|------|------|
| 세그먼트 | NPL내부자 / HNWI / 엔젤 / SaaS고객 / 기관LP |
| 예상 티켓 (원) | 1,000만 / 5,000만 / 1억 / 5억 / 10억+ |
| 의향 단계 | 인지 / 관심 / 정보요청 / LOI / 청약완료 |
| 최초 접점 경로 | 네트워크 / SaaS영업 / 세미나 / 소개 |
| 마지막 접촉일 | 날짜 |
| 다음 액션 | 미팅 / IM 발송 / LOI 회수 / KYC 안내 |
| 메모 | 담보 선호, 우려사항, 의사결정권자 여부 |

**실무 기준:** LOI 20건 이상 = 1호 발행 트리거. 그 아래면 발행 일정 재조정.

## 4. SaaS ↔ 앵커투자자 교차전환 플라이휠 (핵심)

### 4.1 플라이휠 구조

CEO 통찰: "SaaS 고객이 곧 앵커투자자 후보다." 이 고리를 명시적으로 설계한다.

```
[평가 SaaS 영업]
    │
    ├─ NPL 투자사·AMC·법무법인이 구독
    │      │
    │      └─ 실거래 데이터·회수 사례가 플랫폼에 축적됨
    │
    ▼
[데이터 정밀도 향상]
    │
    └─ p10~p90 백테스트 오차 감소 → "이 플랫폼의 IRR 예측을 믿을 수 있다"
    
    ▼
[SaaS 고객 → 토큰 청약자 전환]
    │
    ├─ "당신이 매일 쓰는 플랫폼이 평가한 물건입니다"
    │   → 구조 설명 시간 절감 + 신뢰 기형성
    │
    └─ SaaS 고객사 담당자가 내부 LP에게 소개 → 기관 청약 경로 개통
    
    ▼
[토큰 발행 완주 + 회수 실적 누적]
    │
    └─ 실제 회수 데이터 → SaaS 모델 정확도 추가 향상
       → 다음 발행 IRR 신뢰도 ↑ → 더 많은 SaaS 고객·청약자 유입

→ 루프 반복
```

**한 줄 요약:** SaaS로 데이터 신뢰를 쌓고, 그 신뢰가 청약을 부르고, 청약 완주가 다시 SaaS 정확도를 높인다.

### 4.2 교차전환 KPI

| KPI | 기준값 | 목표 (6개월) |
|-----|--------|------------|
| SaaS 월간 구독 고객 수 | 0 | 5개사 |
| SaaS 고객 중 LOI 전환율 | — | 30% (5개사 중 1~2개사) |
| SaaS 경유 청약 비중 | — | 1호 발행 청약액의 20%+ |
| 회수 완료 후 재청약율 | — | 1호 청약자의 50%+ (2호 기준) |

### 4.3 SaaS 영업 시 투자자 전환 준비 체크

SaaS 계약 협상 단계에서 아래를 자연스럽게 포함시킨다:
- 계약서에 "베타 투자자 프로그램 참여 우선권" 조항 추가 (옵션)
- 담당자에게 IM(투자안내문) 수신 동의 여부 사전 확인
- 사용 데이터가 토큰 발행 물건 선정에 어떻게 연결되는지 1페이지 설명 자료 준비

## 5. 실행 로드맵

### 0~3개월 — 수요 파이프라인 기반 구축 (내부 액션)

| 액션 | 담당 | 비고 |
|------|------|------|
| 기존 NPL 거래 관계자 명단 정리 (CRM 입력) | 내부 | T0 — 즉시 시작 |

| SaaS 영업 피치에 "베타 투자자 프로그램" 문구 추가 | 내부 | T0 |
| 수도권 아파트 담보 NPL 1호 풀 물건 선별 (내부 평가) | 내부 | T0 |
| LOI 양식 초안 작성 | 내부 | T0 |

| **[T2] 기존 거래 관계자 1:1 미팅 (의향 타진)** | 대표 컨펌 | 외부 접촉 |

| 액션 | 담당 | 비고 |
|------|------|------|
| **[T2] NPL 특화 소규모 세미나 개최** (20~30명, 비공개) | 대표 컨펌 | 외부 이벤트 |
| **[T2] SaaS 고객사 담당자 청약 의향 전환 미팅** | 대표 컨펌 | 외부 접촉 |
| LOI 수집 (목표: 20건, 총 예상 규모 30억+) | 내부 관리 | T2 접촉 결과 |
| 법무 의견서 수령 후 1호 발행 구조 확정 | 내부 | T2 선행 |

| KYC/AML 프로세스 설계 (내부) | 내부 | T0 |

### 6~12개월 — 1호 발행 실행

| 액션 | 담당 | 비고 |
|------|------|------|
| **[T2] 증권신고서 제출 또는 사모 청약 개시** | 대표 컨펌 | 법무 완료 후 |
| **[T2] 앵커 투자자 청약 완료 (선청약)** | 대표 컨펌 | 외부 자금 수령 |
| **[T2] 나머지 청약 오픈 (Fill)** | 대표 컨펌 | |
| 회수 모니터링 + 투자자 분기 리포트 발송 | 내부 | 토큰 홀더 대상 |
| 1호 회수 완료 → 2호 발행 준비 | 내부 + T2 | 회수 실적 데이터로 IRR 백테스트 갱신 |

## 6. T2 컨펌 필수 항목 요약

> 아래는 **외부 접촉·자금 수령·법인 설립이 포함된 행위**다. 내부 준비만으로 실행 불가.  
> 대표님 확인 후 실행한다.

| # | 항목 | 카테고리 | 선행 조건 |
|---|------|----------|-----------|
| T2-① | 법무법인 증권성 판단 의뢰 | EXTERNAL | 1호 발행 구조 내부 확정 |
| T2-② | 기존 NPL 거래 관계자 1:1 의향 타진 미팅 | EXTERNAL | CRM 명단 정리 완료 |
| T2-③ | NPL 특화 소규모 세미나 개최 (20~30명) | EXTERNAL | IM 초안 + LOI 양식 준비 |
| T2-④ | SaaS 고객사 청약 전환 미팅 | EXTERNAL | SaaS 계약 진행 중인 고객사 특정 |
| T2-⑤ | 엔젤클럽·적격투자자 네트워크 발표 | EXTERNAL | 1호 풀 구성 내부 확정 후 |
| T2-⑥ | SPC 또는 신탁 법인 설립 | EXTERNAL + DIRECTION | T2-① 법무 의견서 수령 후 |
| T2-⑦ | 청약 개시 (LOI → 실제 자금 수령) | EXTERNAL + SECURITY | 법무 완료 + LOI 20건 이상 |
| T2-⑧ | 혁신금융서비스 샌드박스 신청 제출 | EXTERNAL | T2-①②③ 완료 후 |

## 7. 솔직한 수요측 약점 평가

| 약점 | 강도 | 현실 |
|------|------|------|
| NPL 구조 이해 투자자 모수 자체가 적다 | 높음 | 수도권 아파트 담보라도 "경매 회수"라는 개념을 이해하는 개인 투자자는 소수. 1호 발행 50인 채우기도 쉽지 않다. |
| 앵커 투자자 확보 전에는 일반 청약이 안 열린다 | 높음 | 앵커 없이 공개 청약하면 "0명 청약" 리스크 존재. 앵커 확보가 발행의 gate다. |
| 사모 50인 제한 vs. 의미 있는 규모 충돌 | 중간 | 20~40명으로 40억 채우려면 인당 평균 1억 이상. 이 규모를 낼 수 있는 네트워크가 있어야 한다. |
| SaaS 고객 전환은 SaaS 먼저 팔아야 가능 | 중간 | SaaS 구독 고객이 없으면 플라이휠이 돌지 않는다. SaaS 영업과 투자자 파이프라인은 병렬 실행해야 한다. |
| 규제 완료 전 LOI의 법적 효력 불명확 | 낮음 | LOI는 비구속적이므로 투자자가 최종 단계에서 이탈할 수 있다. 규제 확정이 늦어지면 LOI 유효기간 내 발행이 안 될 수 있다. |
| 지방 토지 비중 높은 파이프라인 | 낮음 | 7.7조 중 수도권 아파트 담보 물건이 얼마나 되는지가 중요. 내부 파이프라인 재분류 필요. |

**결론:** 수요 파이프라인은 지금 없다. 그러나 "없다"는 게 "만들 수 없다"는 뜻은 아니다.  
NPL 업계 내부자 10~20명 + 기존 HNWI 네트워크로 1호를 완주할 수 있는 규모(20~40억)는  
현실적으로 확보 가능하다. 핵심은 **첫 2~3명의 앵커를 누구로 할 것인지 지금 결정하는 것**이다.
