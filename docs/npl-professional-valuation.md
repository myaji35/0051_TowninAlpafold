# NPL 전문 평가 체계 설계 — 공공데이터 기반

> 작성: 2026-06-12 · 배경: 현재 평가가 "고정 낙찰가율 0.82 + 권리관계 차감"으로 빈약.
> 목표: 온나라 부동산정보 · 건축물대장 · 경매 실적 등을 동원한 **실무 수준 NPL 평가**.

## 0. 현재의 한계 (정직한 진단)

```python
# backend/npl_scorer.py — 지금
AUCTION_RATE = {"p10": 0.68, "p50": 0.82, "p90": 0.95}   # 전 물건 동일 고정값
net = appraisal * 0.82 - (선순위 + 세금 + 보증금)          # 차감만
```
문제: **모든 물건이 같은 낙찰가율**을 쓴다. 강남 아파트와 지방 토지가 동일 0.82.
실제 NPL 평가는 물건별 특성(지역·유형·연식·권리·시장)으로 낙찰가율 자체가 달라진다.

## 1. 실무 NPL 평가가 동원하는 데이터 (대표님 지적 반영)

| 데이터 | 출처 | 평가 기여 | 연동 가능성 |
|---|---|---|---|
| **공시지가** | 국토부 (molit) | 담보가 기준선 | ✅ **이미 ETL 있음** (data_raw/molit_landprice) |
| **실거래가** | 국토부 실거래가 공개 API | 시세 추정 (감정가 보정) | 🟡 공공 API 존재 (apis.data.go.kr) |
| **건축물대장** | 정부24/건축HUB API | 연식·구조·면적·위반건축물 → 감가 | 🟡 공공 API 존재 (건축HUB) |
| **온나라 부동산정보** | 국토부 부동산종합공부 | 토지+건물+가격 통합 열람 | 🔴 기관용/제한적 (개별 열람) |
| **경매 낙찰 실적** | 법원경매(대법원) / 민간(굿옥션 등) | 낙찰가율 분포 (지역·유형별) | 🟡 대법원 공개 일부 / 민간 유료 |
| **등기부 권리관계** | 인터넷등기소 | 선순위·가압류·임차권 | 🔴 건당 유료 열람 |
| **생활인구/상권** | KOSIS/소상공인 | 입지 가치 (상업용) | ✅ **이미 ETL 있음** |

→ **핵심 통찰**: 공시지가·생활인구는 이미 있다. 실거래가·건축물대장은 **공공 API로 연동 가능**.
   등기부·온나라 통합열람은 건당 유료/기관용 → **수동 입력 또는 별도 계약** 필요.

## 2. 전문 평가 모델 — 낙찰가율을 "동적 산출"로

고정 0.82 → **물건 특성 기반 동적 낙찰가율**:

```
낙찰가율(물건) = 기준낙찰가율(지역×유형)
              × 연식보정(건축물대장)
              × 권리위험보정(선순위/임차)
              × 시장보정(실거래가 추세)
```

### 2.1 기준 낙찰가율 (지역 × 유형) — 경매 통계 기반
현재 단일값 대신 **지역×담보유형 행렬**. 예(수도권 아파트 ~85%, 지방 토지 ~62%):

| 유형 \ 지역 | 수도권 | 광역시 | 지방 |
|---|---|---|---|
| 아파트 | 0.86 | 0.80 | 0.74 |
| 오피스텔 | 0.78 | 0.72 | 0.65 |
| 상가 | 0.68 | 0.62 | 0.55 |
| 토지 | 0.70 | 0.64 | 0.58 |

(출처: 대법원 경매 통계 / 민간 낙찰가율 — 분기 갱신. 초기값은 공개 통계로 시드)

### 2.2 건축물대장 보정 (연식·구조·위반)
- 연식: 신축(0~5년) ×1.05 · 노후(30년+) ×0.85
- 구조: 철근콘크리트 ×1.0 · 조적/목조 ×0.9
- **위반건축물** 등재 시 ×0.7 (NPL 핵심 리스크 — 실무에서 반드시 확인)

### 2.3 실거래가 보정 (감정가 신뢰도)
- 감정가 대비 최근 실거래 中위가 ±15% 이내 → 감정가 신뢰 (confidence ↑)
- 실거래 < 감정가 × 0.85 → 감정가 과대평가 의심, 보수 적용 + confidence ↓

### 2.4 권리관계 정밀화 (현행 + 강화)
- 선순위 + 조세채권(최우선) + 임차보증금(대항력)
- **소액임차인 최우선변제** 반영 (지역별 한도)
- 가압류/가처분 등재 시 회수 지연 → recovery_months ↑

## 3. 단계별 구현 (현실적 연동 순서)

| Phase | 범위 | 데이터 | 비고 |
|---|---|---|---|
| **V1** | 지역×유형 낙찰가율 행렬 | 공개 경매 통계 시드 | 즉시 — 고정값→행렬 |
| **V2** | 건축물대장 연동 | 건축HUB 공공 API | 연식·구조·위반 보정 |
| **V3** | 실거래가 연동 | 국토부 실거래 공공 API | 감정가 신뢰도 보정 |
| **V4** | 소액임차/조세 정밀 권리 | 지역별 한도표(공개) | 수동입력 보완 |
| **V5** | 등기부/온나라 | 건당 유료·기관계약 | 수동입력 또는 별도 계약 |

→ **V1~V4는 공공데이터/공개통계로 자체 구현 가능.** V5(등기부·온나라 통합열람)는
   건당 과금/기관 권한이라 — 수동 입력 필드로 받거나 클라이언트 계약 후 연동.

## 4. 신뢰도(confidence)의 의미 강화

지금은 0.60 고정. 전문 평가에선 **데이터 충족도 = 신뢰도**:
```
confidence = base
  + 건축물대장 연동됨? +0.15
  + 실거래가 3건 이상? +0.15
  + 등기부 확인됨?     +0.10
  − 위반건축물/권리하자? −0.20
```
→ "신뢰도 90% = 4개 소스 교차검증된 평가" vs "신뢰도 45% = 입력값만, 실사 필요".
   이것이 객관성의 핵심 — **평가가 얼마나 많은 근거로 뒷받침되는지** 정량화.

## 5. 기존 자산 재활용

- molit_landprice ETL: **공시지가 → 담보가 기준선** 즉시 활용
- kosis/localdata ETL: **상업용 입지 가치** 보정
- backend/etl 인프라: 건축물대장·실거래 ETL 추가는 같은 패턴
- npl_scorer.py: 낙찰가율을 상수→함수로 교체 (구조 유지)

## 6. 미결정 (대표님 확인)

1. **공공 API 키** — 건축HUB·국토부 실거래 API는 data.go.kr 인증키 필요. 발급/등록은 T2(외부).
2. **등기부·온나라** — 건당 유료. 자동 연동 vs 수동 입력 vs 클라이언트 계약 중 선택.
3. **낙찰가율 행렬 출처** — 공개 대법원 통계로 시드 vs 민간 데이터(유료) 도입.
4. **우선순위** — V1(행렬)부터 점진 vs 특정 소스(예: 건축물대장) 우선.
